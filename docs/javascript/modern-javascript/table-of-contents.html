🚀 목차

📚 책 정보​

  • 도서 정보
  • 저: 조 모건

✌️ Chapter 1: 변수 할당으로 의도를 표현하라​

  1. const로 변하지 않는 값을 표현하라.
  2. let과 const로 유효 범위 충돌을 줄여라.
  3. 블록 유효 범위 변수로 정보를 격리하라.
  4. 테믈릿 리터럴로 변수를 읽을 수 있는 문자열로 변환하라.

✌ Chapter 2: 배열로 데이터 컬렉션을 관리하라​

  1. 배열로 유연한 컬렉션을 생성하라.
  2. includes()로 존재 여부를 확인하라.
  3. 펼침 연산자로 배열을 본떠라.
  4. push() 메서드 대신 펼침 연산자로 원본 변경을 피하라.
  5. 펼침 연산자로 정렬에 의한 혼란을 피하라.

✌ Chapter 3: 특수한 컬렉션을 이용해 코드 명료성을 극대화하라​

  1. 객체를 이용해 정적인 키-값을 탐색하라.
  2. Object.assign()으로 조작 없이 객체를 생성하라.
  3. 객체 펼침 연산자로 정보를 갱신하라.
  4. 맵으로 명확하게 키-값 데이터를 갱신하라.
  5. 맵과 펼침 연산자로 키-값 데이터를 순회하라.
  6. 맵 생성 시 부수 효과를 피하라.
  7. 세트를 이용해 고윳값을 관리하라.

✌ Chapter 4: 조건문을 깔끔하게 작성하라​

  1. 거짓 값이 있는 조건문을 축약하라.
  2. 삼항 연산자로 빠르게 데이터를 확인하라.
  3. 단락 평가를 이용해 효율성을 극대화하라.

✌ Chapter 5: 반복문을 단순하게 만들어라​

  1. 화살표 함수로 반복문을 단순하게 만들어라.
  2. 배열 메서드로 반복문을 짧게 작성하라.
  3. map() 메서드로 비슷한 길이의 배열을 생성하라.
  4. filter()와 find()로 데이터의 부분집합을 생성하라.
  5. forEach()로 동일한 동작을 적용하라.
  6. 체이닝으로 베서드를 연결하라.
  7. reduce()로 배열 데이터를 변환하라.
  8. for...in 문과 for...of 문으로 반복문을 정리하라.

✌ Chapter 6: 매개변수와 return 문을 정리하라​

  1. 매개변수 기본값을 생성하라
  2. 해체 할당으로 객체 속성에 접근하라
  3. 키-값 할당을 단순화하라
  4. 나머지 매개변수로 여러 개의 인수를 변수로 전달하라

✌ Chapter 7: 유연한 함수를 만들어라​

  1. 테스트하기 쉬운 함수를 작성하라
  2. 화살표 함수로 복잡도를 낮춰라
  3. 부분 적용 함수로 단일 책임 매개변수를 관리하라
  4. 커링과 배열 메서드를 조합한 부분 적용 함수를 사용하라
  5. 화살표 함수로 문맥 혼동을 피하라

✌ Chapter 8: 클래스로 인터페이스를 간결하게 유지하라​

  1. 읽기 쉬운 클래스를 만들어라
  2. 상속으로 메서드를 공유하라
  3. 클래스로 기존의 프로토타입을 확장하라
  4. get과 set으로 인터페이스를 단순하게 만들어라
  5. 제너레이터로 이터러블 속성을 생성하라
  6. bind()로 문맥을 해결하라

✌ Chapter 9: 외부 데이터에 접근하라​

  1. 프라미스를 이용해 비동기적으로 데이터를 가져오라
  2. async/await로 함수를 명료하게 생성하라
  3. fetch로 간단한 AJAX 호출을 처리하라
  4. localStorage로 상태를 장기간 유지하라

✌ Chapter 10: 컴포넌트 아키텍처를 이용해 관련 파일을 모아라​

  1. 가져오기와 내보내기로 기능을 분리하라
  2. npm으로 커뮤니티 코드를 끌어와라
  3. 컴포넌트 아키텍처를 이용해 애플리케이션을 만들어라
  4. 빌드 도구를 이용해 컴포넌트를 결합하라
  5. CSS 애니메이션을 활용하라
